  <name>Task 2: Create servista.library base convention plugin with code quality and directory validation</name>
  <files>

  <action>
    **src/main/kotlin/servista.library.gradle.kts:**
    This is the base convention plugin. All other plugins will apply this. It provides:

    1. **Kotlin/JVM configuration:**
       - `plugins { kotlin("jvm") }` -- applied from the kotlin-gradle-plugin dependency
       - `plugins { id("dev.detekt") }` -- from detekt-gradle-plugin dependency
       - `plugins { id("com.ncorti.ktfmt.gradle") }` -- from ktfmt gradle plugin dependency
       - `kotlin { jvmToolchain(21) }` -- JVM 21 target per ADR-017

    2. **Directory structure validation:**
       - Register a `validateProjectStructure` task (group = "verification")
       - Check for existence of: `src/main/kotlin`, `src/main/resources`, `src/test/kotlin`
       - Throw `GradleException` listing missing directories if any are absent
       - Wire `compileKotlin` to `dependsOn("validateProjectStructure")`
       - Per user decision: "Build fails if structure is wrong"

    3. **detekt configuration:**
       - `detekt { buildUponDefaultConfig = true; parallel = true }`
       - Load the shared detekt.yml from classpath resources. Use:
         ```
         detekt {
             config.from(
                 resources.text.fromUri(
                     Thread.currentThread().contextClassLoader
                         .getResource("detekt/detekt.yml")!!.toURI()
                 )
             )
         }
         ```
         If this approach fails at runtime (resource not available in precompiled script context), fall back to `config.setFrom(files("src/main/resources/detekt/detekt.yml"))` with a comment noting the limitation.

    4. **ktfmt configuration:**
       - `ktfmt { kotlinLangStyle() }` -- Kotlin official style guide formatting

    5. **Common Kotlin dependencies (minimal):**
       - `dependencies { implementation("org.jetbrains.kotlinx:kotlinx-coroutines-core:1.10.2") }`
       - `dependencies { implementation("org.jetbrains.kotlinx:kotlinx-datetime:0.7.1") }`
       - These are the baseline dependencies every Kotlin project needs. Version numbers match ADR-017 and the published catalog.

    **src/main/resources/detekt/detekt.yml:**
    Create a conservative detekt configuration. Since we are using detekt 2.0.0-alpha.2 (which renamed rule categories), use the NEW category names:
    - `build: maxIssues: 0` -- fail on any issue (strict)
    - `comments: active: true` -- documentation rules
    - `complexity: active: true` with reasonable thresholds (e.g., `LongMethod: threshold: 60`, `ComplexMethod: threshold: 15`, `LargeClass: threshold: 600`)
    - `coroutines: active: true`
    - `emptyblocks: active: true` -- was "empty" in detekt 1.x
    - `exceptions: active: true`
    - `naming: active: true`
    - `performance: active: true`
    - `potentialbugs: active: true` -- was "bugs" in detekt 1.x
    - `style: active: true`
    - Keep it minimal and permissive initially. We can tighten later. The important thing is that detekt runs and catches obvious issues.

    After creating both files, verify the plugin compiles:
    - Run `./gradlew classes` to compile the precompiled script plugin
    - Check `build/classes/kotlin/main/` to confirm the generated class name matches what is registered in the `gradlePlugin {}` block in build.gradle.kts
    - If the generated class name differs from `Servista_libraryPlugin`, update build.gradle.kts accordingly
